# 跨境电商亏损计算系统 — 财务模型分析报告

## 1. 概述

### 1.1 系统定位

本系统是一套面向 **Wildberries（WB）平台中国卖家** 的跨境电商利润计算与折扣推算工具。系统覆盖跨境店铺（CNY 计价）和本土店铺（RUB 计价）两种业务模式，支持从 Excel 导入商品数据、自动匹配 SKU 和佣金率、批量计算盈亏平衡点，并基于利润目标导出调价策略。

### 1.2 核心方法

系统采用 **CVP（Cost-Volume-Profit，本量利）模型** 作为计算基础。CVP 模型将成本分解为固定成本和变动成本，通过公式推导出盈亏平衡点（保本价），进而计算目标利润下的目标售价和折扣空间。

核心逻辑可概括为：

```
收入 - 变动成本 - 固定成本 = 利润
```

其中变动成本以售价的百分比形式表达（综合费率 R_Total），固定成本则聚合了所有与售价无关的绝对成本项（运费、产品成本、退货风险等）。

### 1.3 适用场景

- 商品上架前的定价测算
- 在售商品的折扣与调价策略
- 批量导出 Excel 结果供运营执行
- 跨店铺（跨境/本土）统一测算口径

## 2. 核心计算公式

### 2.1 体积与运费计算

运费基于商品体积（升）通过分段函数计算。体积升数公式：

```
volume_L = L × W × H / 1000
```

其中 L、W、H 为商品包装尺寸，单位为厘米。

**跨境运费（WB 跨境-FBS，国内发货）：**

```
shipping = base_fee + max(0, ceil(volume_L) - 1) × rate_per_unit
```

默认值：`base_fee = 8.0`（首升），`rate_per_unit = 2.0`（续升费率），单位为 CNY。

等价于：

```
shipping = 8 + max(0, ceil(V) - 1) × 2
```

**本土运费（WB 本土-FBS，本土发货）：**

```
if volume_L <= 1:
    shipping = 32.0
else:
    shipping = 46 + (ceil(volume_L) - 1) × 14
```

单位：RUB。首升 32 RUB，超出部分每升 14 RUB，且 1L 以上起征点为 46 RUB。

> 注：系统预留了 FBW（中国仓）运费模板，目前尚未启用。

---

### 2.2 成本分解

#### 基础成本 C_base

基础成本聚合了所有与商品直接相关的物流和操作费用，经汇率风险系数调整后，再按货损率放大。

```
C_base = (product_cost + dropship_fee + pack_fee + shipping + scan_fee) × risk_rate / (1 - damage_rate / 100)
```

各项含义：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| product_cost | — | 产品成本（从飞书同步的分销价转换而来） |
| dropship_fee | 10.0 RUB | 代发费 |
| pack_fee | 0.0 RUB | 包装费 |
| shipping | — | 运费（由体积计算） |
| scan_fee | 1.0 RUB | 平台扫码费 |
| risk_rate | 1.00 | 汇率风险系数 |
| damage_rate | 2.0% | 货损率 |

#### 退货风险成本 C_risk

退货成本分摊到每一件商品上，考虑了往返运费、处理费以及退货后的残值损失。

```
C_risk = return_rate / 100 × (
    dropship_fee + shipping + scan_fee
    + return_pickup_fee + return_process_fee
    + product_cost × residual_loss_rate / 100
)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| return_rate | 15.0% | 退货率 |
| return_pickup_fee | 0.0 RUB | 退货取件费 |
| return_process_fee | 10.0 RUB | 退货处理费 |
| residual_loss_rate | 15.0% | 退货残值损失率（退货后商品价值损失比例） |

#### 综合变动费率 R_Total

R_Total 是所有按售价百分比计算的费用项之和，是 CVP 模型中"变动成本率"的核心指标。

```
R_Total = real_commission + effective_withdraw + ad_percent / 100 + ops_rate / 100 + member_disc / 100
```

其中：

```
real_commission = commission_rate / 100 × (1 - commission_discount / 100)
effective_withdraw = withdraw_fee / 100 × (1 - real_commission)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| commission_rate | 10.0% | 平台佣金率 |
| commission_discount | 0.0% | 佣金优惠减免率 |
| withdraw_fee | 1.0% | 提现手续费率（按扣除佣金后的金额计算） |
| ad_percent | 2.5% | 广告费率 (ACOS) |
| ops_rate | 3.0% | 运营成本率 |
| member_disc | 3.0% | 会员折扣率（平台会员折扣由卖家承担的部分） |

**默认 R_Total 计算示例：**

```
real_commission = 0.10 × (1 - 0) = 0.10
effective_withdraw = 0.01 × (1 - 0.10) = 0.009
ad_percent = 0.025
ops_rate = 0.03
member_disc = 0.03

R_Total = 0.10 + 0.009 + 0.025 + 0.03 + 0.03 = 0.194（即 19.4%）
```

#### 总固定成本 Total_Fixed

将所有固定成本汇总，同时扣除运费部分对应的提现手续费节省：

```
Total_Fixed = C_base + C_risk + ad_fixed - shipping × withdraw_fee / 100
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| ad_fixed | 0.0 RUB | 广告固定支出（不与售价挂钩的固定广告费用） |

> 注意：提现手续费节省项（`- shipping × withdraw_fee / 100`）的含义是：运费不产生平台佣金，因此提现时免除了这部分对应的手续费。

---

### 2.3 盈亏与利润

#### 盈亏平衡点（保本价）

保本价是使利润为零的售价：

```
BreakEven = Total_Fixed / (1 - R_Total)
```

当售价等于保本价时：

```
Profit = BreakEven × (1 - R_Total) - Total_Fixed = 0
```

**默认保本加价倍数：**

```
1 / (1 - R_Total) = 1 / 0.806 ≈ 1.24×
```

即在默认参数下，总固定成本需要乘以约 1.24 倍才能覆盖变动成本损失。

#### 利润公式

```
Profit = price × (1 - R_Total) - Total_Fixed
```

等价于：

```
Profit = price - price × R_Total - Total_Fixed
```

#### 最大可用折扣（无损折扣）

不产生亏损的最大折扣力度（向下取整，上限 95%）：

```
MaxDiscount = floor((1 - BreakEven / current_price) × 100)
MaxDiscount = min(95, MaxDiscount)
```

当 `MaxDiscount < 0` 时表示当前售价已经低于保本价，需要涨价才能止损。

#### 最低保本价

```
MinPrice = max(BreakEven, 5)
```

下限 5 对应 WB 平台最低售价约束（`Config.PRICE_MIN = 5`）。

---

### 2.4 目标计算

#### 目标售价 — 利润率模式

按目标利润率（相对于售价的百分比）计算目标售价：

```
TargetPrice = Total_Fixed / (1 - R_Total - target_profit_rate / 100)
```

推导依据：

```
TargetPrice - TargetPrice × R_Total - Total_Fixed = TargetPrice × target_profit_rate / 100
=> TargetPrice × (1 - R_Total - target_profit_rate / 100) = Total_Fixed
=> TargetPrice = Total_Fixed / (1 - R_Total - target_profit_rate / 100)
```

默认目标利润率：`target_profit_rate = 10.0%`。

#### 目标售价 — 固定利润模式

按每件商品固定利润额计算目标售价：

```
TargetPrice = (Total_Fixed + fixed_profit) / (1 - R_Total)
```

默认固定利润额：`target_profit_amount = 20.0 CNY`。

#### 目标折扣

达到目标售价所需的折扣力度：

```
TargetDiscount = floor((1 - TargetPrice / current_price) × 100)
TargetDiscount = min(95, TargetDiscount)
```

负数表示需要涨价才能达到目标利润。

#### 目标新原价

当保持当前折扣率不变时，需要设定的新原价：

```
TargetNewPrice = TargetPrice / (1 - current_discount / 100)
```

此公式用于"保持折扣调价"策略：例如当前折扣 20%，目标售价 80 元，则新原价 = 80 / (1 - 0.20) = 100 元，使得折后价恰好等于目标售价。

## 4. 俄罗斯电商平台费率对比

### 4.1 WB vs Ozon 费率对比

以下对比基于行业公开数据和系统默认参数：

| 费用项 | 系统默认值 | WB 实际范围 | Ozon 实际范围 | 说明 |
|--------|-----------|------------|-------------|------|
| 平台佣金 | 10% | 10–19%（按品类） | 8–20%（按品类） | WB 服装/鞋帽类较高，电子类较低 |
| 物流配送费 | 体积计费 | 40–50 RUB/L | 类似体积计费 | 系统跨境 8+2×V CNY ≈ 本地计费 |
| 退货率 | 15% | 10–25%（服装类可达 30%+） | 8–20% | 服装鞋帽退货率显著高于电子品类 |
| 仓储费 | 无（FBS 模式无仓储费） | 0.14 RUB/L/天（FBO） | 阶梯递增 | FBO 模式仓储费从第 31 天起递增 |
| 广告费用 (ACOS) | 2.5% | 5–20% 常见 | 5–15% 常见 | 系统默认偏低，实际竞争品类偏高 |
| 综合费率 R_Total | ~19% | ~25–35% | ~25–30% | 含佣金+提现+广告+运营+折扣 |

### 4.2 FBO vs FBS 模式对比

| 维度 | FBO（平台仓储配送） | FBS（自发货） |
|------|-------------------|-------------|
| 仓储费 | 有（按升/天计费） | 无 |
| 配送费 | 由平台统一收取 | 按系统运费公式计算 |
| 库存灵活性 | 需提前入仓 | 灵活，可多仓发货 |
| 适用场景 | 爆款/高周转商品 | 长尾/低周转/新品 |
| 系统支持 | 未单独建模（仓储费缺失） | ✓ 完整支持 |

> 当前系统主要针对 FBS 模式设计，缺少 FBO 模式下的仓储费字段，使用者如需测算 FBO 模式应手动在 ad_fixed 中估算仓储成本。

### 4.3 佣金匹配流程

系统采用两步匹配法获取佣金率：

```
SKU（卖家货号）→ 飞书产品（获取类目）→ 佣金表（按类目匹配）
```

匹配来源优先级：商品级匹配 > 品类级匹配 > 默认佣金率（30%）。

## 5. 经营核算建议

### 5.1 参数优化建议

当前系统默认参数来源于通用假设，实际运营中建议根据以下方向调整：

**R_Total 综合费率：19.4% → 建议 25%（保守估计）**

系统默认的 19.4% 偏乐观。实际运营中，特别是服装、鞋帽等品类，佣金率本身就可能达到 15–19%，加上广告投入（ACOS 5–15%）和会员折扣，综合费率通常在 25–35% 之间。建议根据实际品类调整：

| 品类 | 建议 R_Total |
|------|-------------|
| 电子/数码配件 | 20–25% |
| 家居/日用 | 22–28% |
| 服装/鞋帽 | 28–35% |
| 美妆/个护 | 25–32% |

**退货率：15% → 建议 18–20%**

服装品类的退货率经常超过 25%，即使非服装品类也常在 10–15%。建议对新品类取 18–20% 作为保守估计。

**增加仓储费字段**

系统目前缺少 FBO 模式的仓储费计算。对于使用 FBO 的卖家，建议在`ad_fixed`中手动估算月均仓储成本，或在后续版本中增加独立的仓储费输入字段。

**汇率风险缓冲：risk_rate 1.00 → 建议 1.03–1.05**

CNY/RUB 汇率年波动可达 15–25%。将风险系数设为 1.03–1.05 可在一定程度上缓冲汇率不利变动带来的成本冲击。

**佣金率自动获取**

建议对接 WB/Ozon API 实时获取品类佣金率，减少手工维护佣金表的成本和出错可能。

### 5.2 核心定价公式速查

```
保本价 = Total_Fixed / (1 - R_Total)
```

```
目标利润率售价 = Total_Fixed / (1 - R_Total - 目标利润率)
```

```
折后价 = 原价 × (1 - 折扣率 / 100)
```

```
目标折扣（折扣优先策略） = floor((1 - 目标售价 / 当前原价) × 100)
```

```
新原价（保持折扣调价策略） = 目标售价 / (1 - 当前折扣率 / 100)
```

### 5.3 风险提示

**汇率风险**

CNY/RUB 汇率年波动幅度在 15–25% 之间。例如 2022–2023 年间 RUB 兑 CNY 汇率曾出现超过 30% 的波动。系统虽然设有 risk_rate 参数，但该参数是静态的，建议：

- 每月至少校准一次汇率
- 为高利润商品设置更高的风险缓冲
- 关注地缘政治事件对卢布汇率的冲击

**佣金率动态调整**

WB 和 Ozon 的品类佣金率并非一成不变。平台可能根据竞争状况、季节、品类策略调整佣金比例。建议：

- 定期（至少每季度）更新佣金表
- 关注平台公告中的费率变动通知
- 将 "佣金未匹配" 的默认值从 30% 调低（建议 15–20%），因为未匹配通常是新品类或有特殊协议

**退货成本 — 高利润品类的隐形杀手**

对于高单价商品，退货的残值损失（residual_loss_rate = 15%）可能造成远超预期的成本。例如：

- 售价 5000 RUB 的商品退货一次，残值损失 = 5000 × 15% = 750 RUB
- 加上往返运费、处理费，单次退货成本可能超过 1000 RUB
- 如果退货率是 20%，每件商品分摊的退货成本 = 1000 × 20% = 200 RUB
- 这部分成本在售价中的占比可能高达 4–5%，不容忽视

**跨境 vs 本土的币种差异**

- 跨境店铺：成本以 CNY 计价，售价以 RUB 计价（通过汇率转换）
- 本土店铺：全链路以 RUB 计价
- 切换店铺模式时务必检查所有成本参数的币种一致性

## 6. 导出策略说明

导出策略决定了如何将计算结果写回 Excel 模板，供运营执行调价。

### 6.1 五种导出策略

#### 折扣优先（discount_only）

仅写入折扣列（L列：新折扣）。系统计算目标折扣后写入 Excel。适用于只需要调整折扣率、不改变原价的场景。

```
写入: new_discount = target_discount
```

负折扣会修正为 0（即不打折）。

#### 价格优先（price_only）

仅写入价格列（J列：新价格）。使用最低保本价作为新价格。适用于需要直接改价、不依赖折扣的场景。

```
写入: new_price = min_price（即保本价）
```

#### 价格 + 折扣（both）

同时写入新折扣和新价格（保本价）。适用于双管齐下的场景。

```
写入: new_discount = target_discount
写入: new_price = min_price
```

#### 保持折扣调价（keep_discount）

- 如果目标折扣为非负数：仅调整折扣，不修改原价
- 如果目标折扣为负数（即当前原价已无法保本）：保持当前折扣率不变，通过抬高原价（写入 target_new_price）来达到目标利润

```
if target_discount >= 0:
    new_discount = target_discount
else:
    new_price = target_new_price  # 保持折扣，抬高原价
```

此策略适合不希望改变折扣率的场景，例如大促期间保持统一折扣力度。

#### 折扣归零调价（zero_discount）

- 如果目标折扣为非负数：仅调整折扣
- 如果目标折扣为负数：将折扣归零（设为 0），同时将新价格设为 target_price（目标售价）

```
if target_discount >= 0:
    new_discount = target_discount
else:
    new_discount = 0
    new_price = target_price
```

此策略适用于希望清理折扣、以实际售价为基准的场景。

### 6.2 导出筛选

所有策略都支持导出筛选，仅将符合条件的商品写入结果：

| 筛选模式 | 说明 | 默认阈值 |
|---------|------|---------|
| 按利润值（profit_value） | 仅导出利润低于指定值的商品 | 0（即仅导出亏损商品） |
| 按利润率（profit_rate） | 仅导出利润率低于指定百分比的商品 | 10% |

筛选逻辑：

```
if mode == "profit_value":
    passes = profit < threshold
elif mode == "profit_rate":
    passes = profit / discounted_price × 100 < threshold
```

只有通过筛选的商品才会被写入新折扣/新价格。

### 6.3 导出流程

```
计算完成（calc_results）
    ↓
读取策略和筛选配置
    ↓
遍历每条结果 → 判断是否通过筛选
    ↓
通过 → 按策略生成新折扣/新价格
    ↓
通过 ExcelService.write_updates() 写入副本文件
    ↓
生成 "_结果.xlsx" 文件
```
